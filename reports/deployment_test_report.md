## Notes on Cold-Start

Per the project KPI definition (Section 9.2 of the proposal), **cold-start latency is excluded** from all measurements. Azure App Service Free Tier idles after 20 minutes of inactivity. The first request after an idle period may take 60–120s as flan-t5 / llama reloads into memory.

The test script sends one warm-up request to `/health` and waits 5 seconds before starting the 20-query test — ensuring all results reflect warm-instance performance.
